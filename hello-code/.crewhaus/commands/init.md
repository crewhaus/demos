Bootstrap a project memory file at the repo root.

Steps:
1. Read `package.json`, `Cargo.toml`, `pyproject.toml`, `go.mod`, and the
   top-level `README.md` (whichever exist) to identify the project's
   purpose and tech stack.
2. Glob the top-level directories. Note which look like `src/`, `tests/`,
   `docs/`, etc.
3. Read 3-5 of the most important files (the entrypoint, a couple of
   core modules) — use the `code-explorer` sub-agent if the repo is
   large.
4. Write `CODE-COMPANION.md` at the repo root with this structure:

   ```
   # CODE-COMPANION.md

   Auto-generated by hello-code /init on <today's date>.
   This file is loaded by future hello-code sessions as project memory.

   ## Project
   <one-line description>

   ## Tech stack
   <one-line summary: language, runtime, frameworks>

   ## How to run
   - Install: <command>
   - Dev:     <command>
   - Test:    <command>
   - Build:   <command>

   ## Codebase map
   - <path>: <one-line role>
   - <path>: <one-line role>
   ... (5-8 entries, top-level only)

   ## Conventions
   <bullets — naming, testing patterns, anything non-obvious>
   ```

5. After writing, print a 2-line summary of what you found and remind the
   user that this file persists across sessions.
